# 搬瓦工 KVM DD 重装系统完整教程：怎么用 KiwiVM 一键重装 Linux？能不能 DD 安装 Windows？普通重装和 DD 有什么区别？

拿到搬瓦工 VPS 的第一件事，很多人都是直奔系统重装。

原因很简单：默认给你装的系统不一定是你想要的，或者你之前折腾坏了，或者就是想从一个干净的环境重新开始。不管哪种情况，搬瓦工的 KVM 架构在这件事上比较省心——它自带一个叫 KiwiVM 的控制面板，把重装系统做成了几步点点的操作。

不过，"重装系统"这件事本身分两种：一种是 KiwiVM 里直接选个 Linux 发行版来装（官方支持的流程）；另一种是用 DD 命令，把一个完整的系统镜像写进磁盘，等于从底层换掉操作系统，Windows 也可以这样装。两个流程完全不同，坑也不一样。

这篇文章把这两条路都说清楚。

## 先搞清楚：搬瓦工 KVM DD 是怎么回事

所谓 DD，是 Linux 里一个用来转换和复制文件的命令，字面意思就是"data duplicator"。用在 VPS 上，它能做到的事是把一个完整的系统镜像（比如 Windows 或者某个定制版 Linux）直接写进 VPS 的硬盘，相当于把原来的系统整个抹掉换新的。

搬瓦工早期用的是 OpenVZ 架构，这种虚拟化方式共用宿主机内核，几乎没有 DD 的可能。**现在搬瓦工已经全面转到 KVM 架构**，KVM 有完整的独立内核，支持自定义系统，DD 就可以操作了。

普通 KiwiVM 重装 vs DD，区别主要在这里：

| 对比维度 | KiwiVM 一键重装 | DD 方式 |
|---|---|---|
| 操作难度 | 低，面板点选 | 中等，需要 SSH 执行脚本 |
| 系统选项 | Linux 官方镜像（Debian/Ubuntu/CentOS） | 几乎任意系统，含 Windows |
| 安全性 | 镜像来自官方，有保障 | 取决于镜像来源是否可信 |
| 数据影响 | 全部清空 | 全部清空 |
| IP 变化 | 不变 | 不变 |
| 适用场景 | 换个 Linux 发行版、恢复干净环境 | 安装 Windows 或高度定制系统 |

两种方式都会把硬盘上的数据清光。这一点不管选哪条路都要提前备份好。

## 方式一：KiwiVM 官方重装系统（推荐新手）

这是最稳的方式，搬瓦工 KVM 套餐买完之后就可以做。

### 操作步骤

**第 1 步：登录 KiwiVM 控制面板**

登录搬瓦工账号，进 Services → My Services，找到你的 VPS，点右边的 KiwiVM Control Panel，就进了控制面板。

**第 2 步：先把 VPS 关机**

进面板之后，主页（Main Controls）会显示当前状态。如果是 Running，先点 Stop，等它完全关机。跳过这步直接重装，会报"Unable to reload a running VPS"的错，让你先停机。

**第 3 步：进 Install new OS**

左侧菜单找到 Install new OS，点进去。

**第 4 步：选系统版本**

系统列表里有 Debian、Ubuntu、CentOS、Rocky Linux 等，还有带 `_bbr` 后缀的版本。几点建议：

- 新手直接选 Ubuntu 24.04 LTS，兼容性最好，教程多
- 想顺手开 BBR 加速，就选带 `_bbr` 后缀的版本，装好就自动开了，不用再手动折腾
- Debian 12 也是很稳的选择，比 Ubuntu 更轻量

带 `_bbr` 的版本为什么值得选：BBR 是 Google 开发的 TCP 拥塞控制算法，跑在 KVM 上能让网络吞吐更稳、丢包更少，建站或者跑服务都有感知。搬瓦工 KVM 套餐直接选带 BBR 的模板，比装完再手动开内核改一遍省事得多。

**第 5 步：勾选条款，点 Reload**

勾上"I agree that all data on my VPS will be lost"，确认你知道数据要没了，然后点 Reload 开始安装。

**第 6 步：记下 SSH 密码**

安装完成后，系统会给你一个新的 root 密码和 SSH 端口。**这个密码只出现一次**，必须马上记下来。忘了的话只能再重置密码，不是什么大问题，但麻烦一道。另外，每次重装 SSH 端口都会变，不是默认的 22，记得去邮件或面板里找新端口。

## 方式二：DD 安装任意系统（进阶操作）

这条路适合有明确需求的人：比如要在搬瓦工 VPS 上跑 Windows，或者要用某个高度定制的 Linux 镜像。

### 前提条件

搬瓦工必须是 KVM 架构才能 DD。目前搬瓦工在售的所有套餐——不管 Basic VPS、CN2 GIA-E 还是香港套餐——全部都是 KVM，这个不用担心。

OpenVZ 老套餐（现在基本没有了）不支持 DD。

### 核心操作流程

1. **先用 KiwiVM 正常安装一个 Linux 系统**（比如 Debian 12），确保能 SSH 进去
2. **通过 SSH 连接到 VPS**，用 root 账号登录
3. **执行 DD 脚本**，将目标系统镜像写入磁盘

DD 的本质是执行一个 shell 脚本，脚本负责下载镜像、写入磁盘、配置引导。脚本跑起来之后，你的 SSH 会话会自动断开，因为磁盘正在被整体替换。

等 10-20 分钟，用 KiwiVM 里的 VNC 功能可以看安装进度。

4. **安装完成后用新系统的密码重新连接**

DD 进去的镜像通常有默认密码，使用前务必改掉。

### 关于 DD Windows 的说明

搬瓦工 KiwiVM 现在本身也支持挂载 ISO 的方式安装 Windows，进面板找 "Mount ISO" 功能，可以自己上传或者挂载可引导镜像，相对比 DD 命令更直观一些。

## 重装过程里几个坑

**坑 1：重装完之后 SSH 死活连不上**

多数原因是端口搞错了。搬瓦工 KVM 重装后 SSH 端口随机生成，不是 22。去 KiwiVM 主页看 SSH port 那一行，用这个端口去连。

**坑 2：数据没备份就点了 Reload**

没有后悔药。搬瓦工提供 Snapshots（快照）和 Automatic Backups（自动备份）功能，重装前先在 KiwiVM 里做一个快照，出了问题还能回滚。

**坑 3：DD 了不可信的镜像**

说实话。网上流传的 DD 镜像质量参差不齐，有些年头比较老，有些来源不明。DD Windows 的时候要用可信来源的镜像，装完第一件事改默认密码，这个真的不能省。

**坑 4：选了 CN2 GIA-E 或者香港套餐却还在纠结 KVM 的问题**

这俩不是同一回事。KVM 是虚拟化架构，所有套餐都是 KVM；CN2 GIA-E 是套餐名称，走的是 CN2 GIA 高端线路，支持 12 个以上机房切换。如果你还在 $49.99/年的 Basic VPS 上，晚高峰网速差是线路问题，重装系统解决不了。

## FAQ

**Q：搬瓦工 KVM 套餐都支持 DD 吗？**

A：支持。目前搬瓦工所有在售套餐都是 KVM 架构，KVM 有完整的独立内核，支持 DD 操作。之前的 OpenVZ 套餐不支持，不过那类套餐基本上已经退出了。

**Q：DD 重装之后 IP 会变吗？**

A：不会。DD 操作只是替换磁盘上的系统，VPS 的 IP 地址不受影响。想换 IP 需要走机房迁移或者单独申请，和重装系统是两件事。

**Q：搬瓦工 KiwiVM 里没有我想要的系统版本怎么办？**

A：两个方向：一是用 KiwiVM 的 "Mount ISO" 功能，可以挂载自定义 ISO 镜像来安装；二是先装一个 KiwiVM 支持的基础系统，再通过 DD 脚本写入你想要的镜像。搬瓦工官方也表示会按需求添加 ISO 镜像，可以提工单申请。

**Q：重装系统之后 SSH 连不上是怎么回事？**

A：十有八九是端口问题。搬瓦工 KVM 重装后 SSH 端口随机变化，不是默认的 22。去 KiwiVM 面板首页找 SSH port 那栏，用新端口去连。其次确认防火墙没有误封，以及连接时用户名是 root。
